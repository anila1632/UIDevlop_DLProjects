Notes on Regular expressions

try this link for regular expression testing

  • A regular expression is a combination of strings wich matches a type of input.Regular exp is of type Object
  • 2 ways of writing regular exp: regular express literal and regular expression constructor.
    regular express literal -> var firstname=/anila/; / / to enclose the exp.
    regular expression constructor -> var firstname=new RegExp("anila");
    Here, in both ex the input the validated char by char first 'a' then 'n' then 'i' and so on.
  • So when to use what?
    when our input value is not changing then use variables and if the value changes or the value comes from other variables then use constructor
  • methods like test() can be used and it returns boolean values. RegExp.prototype.test()
    (/cat/.test("this is cat")) -> willreturn true
    (/cat/.test("this is animal")) -> returns false
  • . -> matches with any single character except line breaks. dot is a wildcard it matches with everything letter digit whitespace. ex: /./
  • ^ -> matches the beginning of a string
  • $ -> matches the end of a string
  • * -> matches the preceding exp 0 or more times
  • + -> matches the preceding exp 1 or more times
  • ? -> preceding exp is optional
  • \d -> matches any single digit character
  • \w -> matches any word character--alphanumeric and underscore
  • [A-K] -> matches any single character in the exp for capital letters in those range
  • [A-K]+ -> matches with atleast one character
  • [ABCD] -> matches with any single character in the brackets
  • [^A-K] -> matches with any single character not uppercase. ^ if used inside acts as negation
  • Flags: there are 5 optional flags which can b used seprately/together and only after closing / ..g-- global search i -- case insensitive search. /[A-Z]/g
  • (x) -> capturing paranthesis. matches x and stores value for x and can use it later.
    (?:x) -> non-capturing.matches x and doesnt store any value
    x(?=y) ->lookahead.matches x only if its followed by y
  • \ -> is backslash... when we have a slash / in the search then use it with backslash.. like (/\//) or for that matter any special charater search should start with backslash and char.. /\$/,/\?/
  • \d=[0-9] digits
  • \w=[A-Za-z0-9_] matches any single char uppercase lowercase digits and underscore

more notes!!

  • Apart from exec(),test() method, other methods are used are replace(),match(),search(),split() methods of String
  • {n} -> matches n occurances of the preceding expression.also {n,} -> matches atleast n times. {n,m} -> n<=m so matches min n times and max m times
  • \d -> matches a digit [0-9] and \D -> matches non-digit ==[^0-9]
    similarly,\w -> matches words a-zA-Z0-9_ and \W matches non-word char other symbols
  • \S -> matches characters other than white-spaces
  • Methods of RegExp: exec and test.
    exec -> executes the search in a string and returns the array of information that is matching or null on mismatch
    test -> tests for a match in a string and returns true/false
  • Methods of String: match,search,replace,split.
    match -> same as exec but it is a string method.
    var searchstring="cat is an animal";var regsrc=/a./;var found=str.match(regsrc);
  • search() -> same like test method but search method returns index of the matched string and -1 if fails
  • replace -> searches for a substring in the given string and replaces with the new substring given.
    str.replace(regexp|substring,newsubstring|function)
  • split -> splits the given string.
    str.split([separator[,limit]]) -- separator can be a regular expression. limit is after how many string the split has to happen
